# Trash Bin Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Soft-delete files into a per-storage trash with restore, permanent purge (including Telegram), empty-trash confirmation, and global retention auto-cleanup (1–30 days, default 30).

**Architecture:** `files.deleted_at` soft-delete; partial unique index for live paths; shared purge helper (Telegram `deleteMessage` then hard DELETE); periodic `TrashPurgeService`; Files-page trash mode + Settings tab for retention.

**Tech Stack:** Rust/Axum/SQLx, SolidJS/SUID UI, Telegram Bot API.

## Global Constraints

- Retention: integer days in `1..=30`, default `30`, max `30`
- Soft delete never calls Telegram; only permanent purge / empty / expiry does
- Trash is per-storage; retention setting is global via `app_settings`
- Follow existing patterns in `FilesRepository`, `ChannelHealthService`, `SettingsModal`, `ActionConfirmDialog`

### Task 1: Schema — `deleted_at` + `app_settings`

**Files:**
- Modify: `sarca/src/startup.rs`
- Modify: `sarca/src/models/files.rs` (add `deleted_at: Option<DateTime<Utc>>` if selected in FromRow queries)

- [ ] **Step 1:** In `init_db` SQL batch, after existing `files` ALTERs, add:

```sql
ALTER TABLE files ADD COLUMN IF NOT EXISTS deleted_at TIMESTAMPTZ;

DO $$
BEGIN
  IF EXISTS (
    SELECT 1 FROM pg_constraint
    WHERE conname = 'files_path_storage_id_key'
  ) THEN
    ALTER TABLE files DROP CONSTRAINT files_path_storage_id_key;
  END IF;
END $$;

CREATE UNIQUE INDEX IF NOT EXISTS files_path_storage_id_alive_uidx
  ON files (path, storage_id)
  WHERE deleted_at IS NULL;

CREATE TABLE IF NOT EXISTS app_settings (
  key   TEXT PRIMARY KEY,
  value TEXT NOT NULL
);

INSERT INTO app_settings (key, value)
VALUES ('trash_retention_days', '30')
ON CONFLICT (key) DO NOTHING;
```
